Data is to be gathered from several sources:

tweet_archive is a dataframe consisting of tweets from the WeRateDogs account. It has each tweet's id as well as some basic string comprehension of the full text. This data is given to us in a csv file.

image_predictions is data containing each tweet's id along with neural net analysis of the associated image, attempting to guess the breed of dog in the image. It is retrieved programmatically from a url offered by Udacity.

tweets_json_df is data scraped from twitter, containing JSON information represented as a dataframe. It contains many relevant properties of each tweet, including the tweet id.

In the previous stage of the project I gathered data from several sources and stored them in the following dataframes:

tweet_archive

image_predictions

tweets_json_df

Each needs to be assessed for quality (at least 8) and tidyiness (at least 2) issues.

Poor Quality, or dirty data, is defined by the following criteria:

1) Completeness (ex: NaN)

2) Validity (ex: negative height, duplicates)

3) Accuracy (ex: valid but wrong)

4) Consistency (ex: NY vs New York)

Untidy, or messy data, is defined by the following criteria:

1) Each variable forms a column

2) Each observation forms a row

3) Each type of observable unit forms a table
